## 明天可以开发什么

按「投入小、收益明显」排序，任选 1～2 项即可。

### 1. Runtime 页体验增强（推荐，约半日）

- **现状**：`/runtime` 已能选主题并 POST 调 `/api/runtime/generate`，脚本跑完返回 stdout/stderr。
- **可做**：
  - **运行中状态**：点击后显示「正在生成…预计 1～5 分钟」，避免重复点击；可选：轮询或简单进度文案（如「已运行 2 分钟」）。
  - **结果展示**：成功时简短提示 +「回首页看新日报」按钮；失败时把 API 返回的 `error` / `stderr` 截取关键几行展示，而不是整块 log。
  - **安全说明**：在 Runtime 页或 About 里加一句：「仅在本机或自建环境使用；部署到公网时不会启用一键触发。」
- **价值**：你已有一键触发，把「等多久、成没成、出错怎么看」说清楚，用起来更放心。

### 2. 全文检索增强（约半日～1 日）

- **现状**：首页已有按关键词搜标题/摘要/简评，即时过滤。
- **可做**：
  - **高亮**：搜索结果里把匹配到的词用 `<mark>` 或 span 高亮显示。
  - **搜索建议**：根据当前输入和已有日报里的高频词，做简单「建议词」或历史搜索（可选，用 localStorage 即可）。
- **价值**：找旧日报更快，体验更「像搜索」。

### 3. 今日简报 / Top N（约半日）

- **现状**：首页有「今天有几份」和按日期的一批卡片，但没有「今日跨主题精选」。
- **可做**：
  - 在首页或单独一块：**今日简报** = 当日所有日报的 highlights 前 1～2 条合并展示（或按主题各取 1 条），可链到对应日报正文。
  - 数据来源：现有 `getAllNewsPreviews` 已能按日期筛，再取每条 `highlights[0]` 即可。
- **价值**：一眼看到「今天所有主题在讲啥」，再决定点进哪份细读。

### 4. 文档与 cron 示例（约 1 小时）

- **可做**：
  - 在 README 或 `docs/s-news.md` 里加一小节：**定时生成**。示例：`crontab -e` 里每天 8 点跑 `run_all_news.sh`（或只跑某几个主题），并注明「需本机已安装并登录 Cursor CLI」。
  - 把上面「App 里点一下调 Cursor CLI」的结论写进文档的「已实现功能」里，避免以后忘。
- **价值**：别人（或半年后的你）clone 后知道「既能手动点 Runtime，也能用 cron 定时跑」。

### 5. 小优化（按需挑）

- **not-found 页**：`app/not-found.tsx` 加一句「回首页」/「按主题浏览」的链接，风格与 About 一致。
- **run_all_news.sh**：确认脚本存在且能依次调各主题（你已有 `run_all_news.sh` 和全部主题脚本，只需确认 API 的 `all` 能正确调用即可）。
- **导出**：今日摘要导出为 Markdown 或纯文本（可选，需求不强可略过）。

---

**建议优先**：先做 **1（Runtime 体验）** 或 **4（文档 + cron）**，再视时间做 **2（高亮）** 或 **3（今日简报）**。
